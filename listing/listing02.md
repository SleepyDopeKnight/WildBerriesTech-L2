Что выведет программа? Объяснить вывод программы. Объяснить как работают defer’ы и их порядок вызовов.

```go
package main

import (
	"fmt"
)


func test() (x int) {
	defer func() {
		x++
	}()
	x = 1
	return
}


func anotherTest() int {
	var x int
	defer func() {
		x++
	}()
	x = 1
	return x
}


func main() {
	fmt.Println(test())
	fmt.Println(anotherTest())
}
```

Ответ:
```
Вывод: 2 1. Это происходит из-за разницы механики возрата именнованных и неименнованных возврщаемых значений.
Именованые возвращаемые значения позволяют менять значение defer'ам, тк они работают с той же переменной,
а неименнованые работают с копией перменной и не меняют ее, тк defer срабатывает после подготовленной для return переменной.

defer откладывает вызов указанной функции до завершения текущей функции, в которой она вызвана.
Вызовы defer складываются в стэк, по этому они вызываются в обратном порядке(Last In First Out) 
```
